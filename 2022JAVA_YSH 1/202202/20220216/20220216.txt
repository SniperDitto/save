

일대 다 관계 / 다대 다 관계 ( 제2정규화 )
고객(엔티티) ->-------<- 상품(엔티티)
고객(엔티티) -|-------<- 판매(행위)
상품(엔티티) -|-------<- 판매(행위)


-- 고객(엔티티) 테이블 : 고객ID, 이름, 성별, 나이
-- 상품(엔티티) 테이블 : 상품ID, 상품명, 중량 
    (재고x, 우리가 파는 상품의 목록같은 개념)
     -> 재고는 함수 이용
     -> 상품에는 단가 넣지말기 : 변동 가능한 행위(입고)의 결과이므로
-- 입고(행위) 테이블 : 순번, 상품ID, 도매상ID, 단가, 입고수량
-- 도매상(엔티티) 테이블 : 도매상ID, 상점이름, 연락처, 주소
-- 판매(행위) 테이블 : 고객ID, 상품ID, 구매가격, 구매수량

-- 고객의 구매 : 상품이 존재해야지만 구매 가능
-- 도매상으로부터의 구매 : 없는 상품을 들고 온다



집계함수 : SUM, COUNT, AVG ...
- ORACLE의 내장 함수 이용
- COL 한 개에 결과를 출력
- 특정 그룹/단체별 결과 확인(GROUP BY)
- 같은 그룹 내에서는 집계함수 여러개 써서 여러 가지 결과 확인 가능
- 잘 이해가 안될때는 엑셀로 만들어서 색칠해보기
- 그룹으로 지정하는 컬럼은 SELECT에 표시하기
    (SELECT COLNAME ~~ GROUP BY COLNAME)


JOIN(합집합)
- 두 테이블을 공통된 필드 기준으로 합쳐서 표시
- 행위로 인해 발생한 엔티티의 모든 속성들을 표시할 수 있다
- 기준이 되는 값에 포함된 데이터의 다른 속성들도 데려올 수 있다
(SELECT * FROM TABLENAME T1, TABLENAME2 T2 WHERE T1.ID=T2.ID)
